LogIn
Description:
This function simply shows the Log In form that the user has to fill up.
Then, the user is taken to different pages, depending on the information given.
Use:
Given that the user is already registered, he/she will find a form where the following data is required:
Email Address: hola@gmail.com
Password: hola123
and the password again, as a confirmation. Once it is filled, the user clicks on the button given to log in.
SignUp
Description:
This function simply shows the Sign Up form.
Then, the function 'NewUser' takes place.
Use:
The user needs to fill up the form given in order to sign up, entering the following data:
Name: Tom
Surname: Cruise
Phone number: 111111111
Email Address: tom@gmail.com
Password: a3
and the password again, as a confirmation. Once it is filled, the user clicks on the button given to sign up.
NewUser
Description:
This function receives all the information provided by the user when signing up.
Then, the function that inserts this new user in the database is called.
Afterwards, depending on the values entered by the user, this user is taken to different pages.
Use:
Once the data entered is reviewed, depending on whether the password entered is the same as the one entered as a confirmation or not, the user is taken to different pages.
If the passwords are the same: The user is given a button that leads to the main user page.
If not: The user is informed that something went wrong and given a button that leads back to the sign up form.
NewUserData
Description:
This servlet contains the functions that insert and delete users in the database. They are called from the servlet called 'NewUser'.
Then, the function that inserts this new user in the database is called.
Afterwards, depending on the values entered by the user, this user is then taken to different pages.
Use:
As far as the database is concerned, there are two functions: The first one inserts the new user into the database and the second one deletes the user that was registered but whose password and its confirmation were not the same.
CheckLogin
Description:
This function receives all the information provided by the user when logging in.
Afterwards, if no errors were made, depending on the values entered by the user, this user is then taken to different pages.
Use:
Once the data entered is reviewed, depending on whether the password entered is the same as the one entered as a confirmation or not, the user is taken to different pages.
If the passwords are the same: The user is given a button that leads to the main user page.
If not: The user is informed that something went wrong and given a button that leads back to the log in form.
If the information entered (email and password) by the user is in the database, meaning he/she is signed up:
The user is given a button that leads to the main user page.
If not: The user is informed about the error and given a button that leads back to the log in form.
However, if the information is correct and the email entered is admin@admin.com:
The user is recognised as an administrator and shown the button that leads to the administration page.
If the information is not in the database:
The user is suggested that maybe he/she is not registered and given a button that leads to the Sign Up form, as well as a button that leads to the Log In form.
CheckLoginData
Description:
This servlet contains the function that gets the password associated to the email entered by the user.It is called from the servlet called 'CheckLogin'.
Use:
As far as the database is concerned:
This function retrieves the password that is associated to the email given and returns it, as well as the email.
